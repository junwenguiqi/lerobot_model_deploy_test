# Dataset Pipeline 改进方案：Raw 采集、LeRobot v3 导出与部署

## 1. 最终建议
推荐架构：

```text
训练pipeline：
实时传感器
  -> SensorBuffer + TimeSync
  -> ObservationBuilder
  -> RawDatasetWriter
  -> raw dataset: PNG + tactile zarr + parquet/jsonl
raw dataset
  -> raw_to_lerobot_v3.py
  -> training dataset: LeRobot v3 风格 parquet + MP4 videos

部署pipeline：
实时传感器
  -> SensorBuffer + TimeSync
  -> ObservationBuilder
  -> OnlinePolicyAdapter
  -> deployed policy
```

核心原则：

> raw 采集保持可靠、可调试；离线导出器负责生成 LeRobot v3 训练数据；部署时不要模拟 LeRobot 文件读取，而是直接构造出和 LeRobot DataLoader 读取后等价的内存 observation。

## 2. 为什么保留 Raw PNG + Zarr 采集

当前 raw dataset 已经保留了很有价值的源数据：

- 全局场景相机 PNG 帧。
- 机械臂腕部相机 PNG 帧。
- 触觉 zarr 数据。
- robot state、command、VR 和同步诊断信息。
- 每帧 timestamp 以及 nearest-neighbor 对齐诊断。

这些信息比在线只写 MP4 更适合调试。如果实时 MP4 编码出现丢帧、颜色格式错误、编码阻塞或时间戳漂移，排查会更困难。PNG + zarr 占空间更大，但作为 raw acquisition format 更稳。

因此建议分成三层：

```text
raw dataset = 原始可靠数据源
LeRobot v3 dataset = 训练 / 回放 / 导出格式
online adapter = 部署输入格式
```

## 3. Raw Dataset 格式

当前 raw layout 基本可以保留：

```text
raw/
  command_events.jsonl
  robot_state_events.jsonl
  vr_controller_events.jsonl

image/
  front/episode_000000/frame_000000.png
  wrist/episode_000000/frame_000000.png

tactile/
  tactile_left/data.zarr
  tactile_left/meta.jsonl
  tactile_right/data.zarr
  tactile_right/meta.jsonl

data/
  chunk-000/file-000.parquet
  episodes/episode_000000.parquet

meta/
  info.json
  episodes.jsonl
  config.yaml
```

## 4. 共享 ObservationBuilder

最重要的代码重构是把“构造一个同步 policy step”的逻辑抽出来。

这个模块应该被两条路径共享：

- 数据采集
- 在线部署

推荐输出 schema：

```python
obs = {
    "timestamp": t,
    "image": {
        "front": front_rgb_uint8,   # H, W, 3
        "wrist": wrist_rgb_uint8,   # H, W, 3
    },
    "state": state_vec,
    "tactile": {
        "left": tactile_left_vec,
        "right": tactile_right_vec,
    },
    "sync": {
        "front_dt": ...,
        "wrist_dt": ...,
        "state_dt": ...,
        "tactile_left_dt": ...,
        "tactile_right_dt": ...,
    },
}
```

采集路径：

```text
SensorBuffer + TimeSync
  -> ObservationBuilder.sample(t)
  -> RawDatasetWriter
```

部署路径：

```text
SensorBuffer + TimeSync
  -> ObservationBuilder.sample(now)
  -> OnlinePolicyAdapter
  -> model inference
```

这样采集和部署会高度一致。模型在线看到的数据，就是同一套同步和 observation 构造逻辑产生的。

## 5. 离线导出：Raw Dataset -> LeRobot v3

新增一个转换脚本：

```text
raw_to_lerobot_v3.py
```

它的职责是：

```text
raw PNG + zarr + parquet/jsonl
  -> LeRobot v3 风格 dataset
```

推荐导出结构：

```text
data/
  chunk-000/file-000.parquet

meta/
  info.json
  tasks.parquet
  episodes/chunk-000/file-000.parquet
  stats.json

videos/
  observation.images.front/chunk-000/file-000.mp4
  observation.images.wrist/chunk-000/file-000.mp4
```

推荐 LeRobot feature names：

```text
observation.images.front
observation.images.wrist
observation.state
observation.tactile_left
observation.tactile_right
action
timestamp
frame_index
episode_index
task_index
```

视觉模态：

```text
observation.images.front: dtype="video"
observation.images.wrist: dtype="video"
```

低维模态：

```text
observation.state: float32[state_dim]
action: float32[action_dim]
```

## 6. Tactile 存储策略

触觉数据建议使用双层存储：

```text
raw layer:
  tactile_left/right 继续存 zarr

training/export layer:
  tactile_left/right 导出成固定形状 parquet columns
```

推荐 LeRobot 训练字段：

```text
observation.tactile_left: float32[234]
observation.tactile_right: float32[234]
```

Paxini 原始触觉帧可以继续在 zarr 中保持 `uint8[234]`。离线导出时，把每个 policy step 对齐后的触觉帧转成训练友好的固定形状数组，例如：

```text
uint8[234] -> float32[234]
```

可选归一化方式：

- `x.astype(float32) / 255.0`
- 使用 dataset statistics 做标准化或 quantile normalization
- 后续按需要拆分 force 和 distribution 字段

当前阶段推荐：

```text
raw zarr 保持原样。
导出 LeRobot v3 时，将对齐后的 tactile frame 写入 parquet 的 observation.tactile_left/right。
```

这样 ACT/DP/OpenPI 风格的 loader 可以把触觉当作普通低维 observation 读取，同时 raw zarr 仍然保留给调试、重处理和未来重新导出。

## 7. 部署输入策略

部署时不要先写 MP4 或 parquet 再推理。

训练路径：

```text
LeRobot MP4/parquet
  -> DataLoader
  -> decoded images + state + tactile
  -> model adapter
  -> model
```

部署路径：

```text
live camera/state/tactile
  -> ObservationBuilder
  -> same semantic observation
  -> model adapter
  -> model
```

部署 adapter 应该构造出和训练 DataLoader 读完 LeRobot 后等价的内存数据：

```python
{
    "observation.images.front": front_rgb_uint8,
    "observation.images.wrist": wrist_rgb_uint8,
    "observation.state": state_vec,
    "observation.tactile_left": tactile_left_vec,
    "observation.tactile_right": tactile_right_vec,
}
```

然后由 model-specific adapter 转成具体模型需要的输入。

例如：

```text
ACT/DP:
  RGB HWC uint8 -> torch tensor，常见为 C,H,W 或 B,T,N_cam,C,H,W

OpenPI:
  RGB HWC uint8 -> OpenPI image dict，然后转为 [-1,1] float32

StarVLA:
  RGB HWC uint8 -> PIL.Image list，然后由 Qwen-VL processor 转成 pixel_values
```

视觉模态在训练和部署中的差异只发生在 semantic observation 之前：

```text
training:
  MP4 -> decoded frame -> observation.images.*

deployment:
  live camera frame -> observation.images.*
```

一旦到达 `observation.images.*` 这一层，训练和部署就应该对齐。

## 8. 实施步骤

推荐按下面顺序推进：

1. 抽出 `ObservationBuilder`。
   - 复用当前 nearest-neighbor sync 逻辑。
   - 输出 RGB HWC uint8 images、state vector、tactile vectors 和 sync diagnostics。

2. 保留 `RawDatasetWriter`。
   - 继续写 PNG、zarr、parquet/jsonl。
   - 明确并统一图像 encoding metadata。

3. 新增 `raw_to_lerobot_v3.py`。
   - 读取 raw episode parquet 和 image index。
   - 将 front/wrist PNG 序列编码成 MP4。
   - 写出 LeRobot 风格 metadata 和 parquet。
   - 将 tactile 导出为固定形状 parquet columns。

4. 新增 `OnlinePolicyAdapter`。
   - 将 `ObservationBuilder` 输出转换为 model-ready input。
   - 按需要分别实现 ACT/DP、OpenPI、StarVLA adapter。

5. 新增 validation script。
   - 比较 raw PNG 和导出 MP4 解码帧。
   - 检查 RGB/BGR 是否正确。
   - 检查 frame count、timestamp alignment、tactile alignment、action/state shapes。

## 9. 最终决策

推荐最终设计：

```text
保留当前 raw 采集：
  PNG images + tactile zarr + raw logs + parquet

新增离线 LeRobot v3 导出：
  MP4 videos + parquet state/action/tactile + LeRobot metadata

新增在线部署 adapter：
  live sensors -> same observation schema -> model-ready tensors
```

这样可以同时获得三个好处：

- 采集和部署强对应。
- 兼容 LeRobot replay，以及 ACT/DP/OpenPI/StarVLA 训练 pipeline。
- 保留完整 raw 数据，方便触觉调试和未来重新导出。
