
# 动作维度
设计为7维空间，采集末端执行器空间+相对增量
observation.state = [
  current_x, current_y, current_z,
  current_rx, current_ry, current_rz,
  current_gripper_width
]  # 7维，绝对当前状态

action = [
  target_x - current_x,
  target_y - current_y,
  target_z - current_z,
  d_rx, 
  d_ry, 
  d_rz,
  target_gripper_width
]  # 7维，相对位姿 + 绝对夹爪

注意两点：
1. d_rx, d_ry, d_rz 表示相对旋转 R_delta 的 rotation vector/axis-angle：
R_delta = R_target * inverse(R_current)
d_rotvec = as_rotvec(R_delta)
不要用欧拉角 RPY，也不要直接用 target_rotvec - current_rotvec。
gripper_width绝对宽度

2. 相对增量的最终语义：step-wise delta
当前推荐把主数据集和 ACT/DP 训练标签定义为 step-wise delta，而不是 chunk-wise delta。

每一行 action_t 表示从当前帧 state_t 到下一帧/下一控制目标 target_{t+1} 的相对增量：

```text
action_t = [
  x_{t+1} - x_t,
  y_{t+1} - y_t,
  z_{t+1} - z_t,
  rot_delta(state_t -> target_{t+1}),
  gripper_width_{t+1}
]
```

ACT/DP 虽然会一次预测一个未来 action chunk，例如 8 步：

```text
chunk_t = [
  action_t,
  action_{t+1},
  action_{t+2},
  ...
  action_{t+7}
]
```

但这个 chunk 只是训练/推理的打包方式。chunk 里的每个元素仍然是一个 step-wise delta：

```text
action_t     = target_{t+1} relative to state_t
action_{t+1} = target_{t+2} relative to state_{t+1}
action_{t+2} = target_{t+3} relative to state_{t+2}
...
```

不推荐把主数据集定义为 chunk-wise delta：

```text
action_t[0] = target_{t+1} relative to state_t
action_t[1] = target_{t+2} relative to state_t
action_t[2] = target_{t+3} relative to state_t
...
```

原因是 chunk-wise delta 会把动作语义绑定到 action_horizon/chunk_size。后续如果 ACT 用 100 步、DP 用 16 步、部署只执行前 1-4 步，数据语义和部署 adapter 都更容易混乱。

3. OpenPI 例外：可单独导出 chunk-wise delta
OpenPI/pi0/pi05 如果启用 `DeltaActions`，它会把未来 action chunk 中每一步都减当前 state：

```text
action_{t+k} = target_{t+k} - state_t
```

这更接近 chunk-wise delta。OpenPI 的这一套语义建议作为单独 exporter/adapter 处理，不改变当前 ACT/DP 主线数据定义。

# 后期工作
1. 采集/保存侧同时转换两类数据
vr控制器发出的命令action、和机器人本体感知state
采集/保存阶段直接落成两类数据
observation.state:
[current_x, current_y, current_z,
 current_rotvec_x, current_rotvec_y, current_rotvec_z,
 current_gripper_width]

action:
[dx, dy, dz,
 d_rotvec_x, d_rotvec_y, d_rotvec_z,
 target_gripper_width]

其中姿态增量不要用 RPY 差值，要用：
R_delta = R_target * inverse(R_current)
d_rotvec = as_rotvec(R_delta)


2. 模型部署
模型由于采用了这个7维且使用相对增量的动作空间，需要反向转换以控制机械臂运动


原始pipeline action/机械臂本体感知：[x, y, z, qx, qy, qz, qw]

3. 改动：
上层保存上一帧 quat
当前帧收到 quat
delta_R = R_curr * R_last.inv()
delta_rotvec = delta_R.as_rotvec()

容器端保持现状：
/getstate 发送当前绝对 pose = [x, y, z, qx, qy, qz, qw]
/pose 接收绝对目标 pose = [x, y, z, qx, qy, qz, qw]

接收数据与保存线程/节点：
1. 从连续两帧 quaternion pose 计算相对增量
   [dx, dy, dz, drx, dry, drz]

2. 控制机械臂命令，即vr端、另引一条线，计算并返回[dx, dy, dz, drx, dry, drz]

3. 如果要控制机械臂（部署），再把相对增量叠加到当前 pose，
   转回绝对目标 pose [x, y, z, qx, qy, qz, qw]，
   发送给 /pose
