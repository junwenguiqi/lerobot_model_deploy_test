
# 各项目视觉模态情况
无论是 LeRobot ACT 还是 LeRobot DP、starvla、还是openpi，模型最终吃到的都不是 MP4、PNG、JPG 文件本身，而是已经读入并预处理后的 torch.Tensor。
更细一点：
如果数据集视觉是 MP4/video：
DataLoader 或 dataset reader 按时间戳从 MP4 解码出对应帧，再转成图像数组/图像 tensor。

如果数据集视觉是 image/PNG/JPG/HF Image：
DataLoader 或 dataset reader 读取图像，再转成图像数组/图像 tensor。

到模型 forward/select_action 前，输入通常已经不再是 MP4/PNG/JPG 文件本身，而是图像数组、PIL 图像，或最终的 tensor。

LeRobot ACT：
多相机常见是 list[tensor]，每个元素一路相机，常见形状 B, C, H, W。

LeRobot DP：
多历史多相机常见整理成 B, T, N_cam, C, H, W。

OpenPI：
多相机 image dict，常见是 B, H, W, C，uint8 会转成 [-1,1] float32，再送入 SigLIP/PaliGemma。

StarVLA：
机器人 VLA 路径常先从视频解码成帧，再转成 PIL.Image list，最后由 Qwen-VL processor 转为 pixel_values tensor。
