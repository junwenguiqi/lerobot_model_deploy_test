# 部署说明

本仓库的实时部署入口只支持：

```text
--policy-kind act
--policy-kind lerobot-dp
```

`DP_train` 和 `openpi_train` 未包含在仓库中，因此 Original DP 和 OpenPI 部署不可用。LeRobot PI0/PI0.5 目前只提供 runtime 类，尚未接入统一部署 CLI。

## 数据链路

```text
ROS 2 相机、机器人状态和触觉消息
  -> live_ros_http_robot_io
  -> canonical observation
  -> policy runtime
  -> action[7]
  -> 安全限幅
  -> ActionTarget
  -> Franka HTTP 服务
```

状态和动作定义：

```text
state  = [x, y, z, rotvec_x, rotvec_y, rotvec_z, gripper_width]
action = [dx, dy, dz, d_rotvec_x, d_rotvec_y, d_rotvec_z, target_gripper_width]
```

`action` 是相对当前末端状态的逐步增量。部署时必须通过 adapter 的 `action_to_target()` 转换，不能把它当成绝对位姿直接发送。

## RobotIO

默认实现是：

```text
policy_adapters/live_ros_http_robot_io.py
```

默认服务地址为占位符 `http://xxx:5000`。运行前应显式设置：

```bash
export FRANKA_SERVER_URL=http://xxx:5000
```

实时部署机器还需要 ROS 2、对应相机话题、机器人状态话题、触觉消息包以及可访问的 Franka HTTP 控制服务。

## 安全要求

1. 第一次运行必须带 `--dry-run`。
2. 检查 front/wrist 是否为 RGB、state 是否为 rotvec、action 方向是否正确。
3. 确认 workspace、位移、旋转、夹爪和频率限制。
4. 每次 episode/reset 后调用 runtime 的 `reset()`。
5. 真机周围无人、急停可用后，才能移除 `--dry-run`。

实际命令见 [运行命令.md](运行命令.md)。
