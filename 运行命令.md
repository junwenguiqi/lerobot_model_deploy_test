# 运行命令

所有命令都从仓库根目录执行。先准备：

```text
/path/to/lerobot_dataset   本地 LeRobot v3 数据集
/path/to/checkpoint.pt     训练生成的 checkpoint
./outputs     本地输出目录
./lerobot/src    LeRobot 0.5.x 源码
```

先确认入口：

```bash
python -m policy_adapters.lerobotACT_train.train_lerobot_act_minimal --help
python -m policy_adapters.lerobotDP_train.train_lerobot_dp_minimal --help
python -m policy_adapters.lerobotPI_train.train_lerobot_pi_minimal --help
python -m policy_adapters.deploy_live_policy --help
```

## ACT

训练：

```bash
python -m policy_adapters.lerobotACT_train.train_lerobot_act_minimal \
  --dataset-root /path/to/lerobot_dataset \
  --repo-id local/fr3_dataset \
  --lerobot-src ./lerobot/src \
  --output-dir ./outputs/act \
  --device cuda \
  --steps 50000 \
  --batch-size 64 \
  --chunk-size 50 \
  --n-action-steps 10 \
  --save-every 500
```

离线评估：

```bash
python -m policy_adapters.lerobotACT_train.offline_eval_lerobot_act \
  --checkpoint /path/to/checkpoint.pt \
  --dataset-root /path/to/lerobot_dataset \
  --repo-id local/fr3_dataset \
  --lerobot-src ./lerobot/src \
  --output-dir ./outputs/act_eval \
  --device cuda \
  --plot
```

## ACT / LeRobot DP 实时部署

先设置机器人服务地址：

```bash
export FRANKA_SERVER_URL=http://xxx:5000
```

ACT dry-run：

```bash
python -m policy_adapters.deploy_live_policy \
  --policy-kind act \
  --checkpoint /path/to/checkpoint.pt \
  --lerobot-src ./lerobot/src \
  --output-dir ./outputs/act_deploy \
  --device cuda \
  --steps 100 \
  --rate-hz 10 \
  --speed-scale 0.25 \
  --dry-run
```

LeRobot DP dry-run：

```bash
python -m policy_adapters.deploy_live_policy \
  --policy-kind lerobot-dp \
  --checkpoint /path/to/checkpoint.pt \
  --lerobot-src ./lerobot/src \
  --output-dir ./outputs/dp_deploy \
  --device cuda \
  --steps 100 \
  --rate-hz 10 \
  --speed-scale 0.25 \
  --dry-run
```

确认日志、观测字段、动作方向和安全边界后，才可以移除 `--dry-run`。

`DP_train` 和 `openpi_train` 未包含在本仓库中，对应命令不可用。
